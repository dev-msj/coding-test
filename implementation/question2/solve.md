# 게임 개발

* 시간 제한: 1초, 메모리 제한: 128MB
* M * N 크기의 맵
* 각각의 칸은 육지 또는 바다
* 각 칸은 (A, B)로 표기
  * A는 북쪽으로부터 떨어진 칸의 개수
  * B는 서쪽으로부터 떨어진 칸의 개수
* 캐릭터는 상하좌우로 이동 가능. 바다는 이동 불가.
* 캐릭터 이동 매뉴얼
  1. 현재 위치 및 방향에서 왼쪽 방향부터 갈 곳을 정함
  2. 왼쪽 방향에 가보지 않은 칸이 있으면 왼쪽으로 회전하여 한 칸 전진
  3. 만약 4면이 모두 가봤거나 바다라면, 바라보는 방향을 유지한 채 뒤로 한 칸 이동 후 1번 수행. 단, 뒤가 바다라면 움직임 종료.
* 캐릭터가 방문한 칸의 수 출력
* 입력
  * 첫번째 줄에 3 <= (N, M) <= 50
  * 두벌째 줄에 좌표 (A, B)와 방향 d 입력
    * 0: 북, 1: 동, 2: 남, 3: 서
  * 이후 각 좌표별 육지(0), 바다(1) 정보 입력
  * 첫 위치는 항상 육지

## 풀이

* `50 * 50 * 4 = 10000`이므로 "O(N)" 복잡도 가능
* A가 북쪽으로부터 떨어진 크기이므로 좌표 이동 시 상하 반전 필요.
  * 북 이동이 (0, -1), 남 이동이 (0, 1)
* move_dict로 방향별 이동 좌표 정의
  
  ```python
  {
    move_dict = {
      0: (0, -1),
      1: (1, 0),
      2: (0, 1),
      3: (-1, 0)
    }
  }
  ```

* d의 크기 증가는 오른쪽 방향으로 회전하므로 `(d - 1 + 4) % 4`로 왼쪽 방향으로 회전하도록 계산
* 이동 정의
  a. `turn_cnt`를 1증가시키고 왼쪽으로 회전
  b. 이동할 좌표의 값이 0이면 이동, 1이면 제자리
  c. 가본 육지는 바다(1)로 변경
  d. `turn_cnt`가 4를 초과하면 3번 이동 수행
  e. `(d - 2 + 4) % 4`를 통해 뒤로 이동할 좌표값을 구함.
  f. 이동 가능하면 다시 1번 과정 수행. 불가능하면 정지.

## 오답 노트

* 풀이 작성 방법이 수도 코드로 작성되지 않아 실제 코드를 작성할 때 코드로 다시 변환해야 해서 시간을 낭비함.
* 좌표의 A가 북으로부터 떨어진 거리라고 했으나 이점을 놓치고 B를 y축 이동으로 사용함.
* 맵 범위를 벗어난 케이스를 고려하지 않음.
* 시작 좌표도 가본 육지로 처리하고 기본 이동 칸을 1로 해야 한다는 것을 놓침.
* 방문 처리를 바다와 동일하게 취급함.
* 맵 크기를 설정할 때 `[행][열]` 구조로 선언해야 했으나 반대로 선언함.
* 3번 조건에 대해서 잘못 이해함. 4면이 다 이동이 불가능해도 아직 가보지 않은 곳이 있을 수 있기 때문에 육지를 통해 뒤로 이동을 반복해야 하는 것을 놓침.

문제의 내용이 많고 풀이가 익숙하지 않은 상태에서 풀이 시간을 40분으로 설정하고 진행하다보니 마음이 급해서 놓친 부분들이 너무 많았다. 풀이 시간을 더 넉넉하게 두고 하나씩 꼼꼼하게 체크하며 풀이하는 연습이 필요하다.
